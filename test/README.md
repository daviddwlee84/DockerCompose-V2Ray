# test/

Local Docker "VPS" for validating Ansible connectivity and a subset of the playbook before deploying to a real cloud VPS.

## What this validates

- SSH reachability from the laptop to the target host (via `just test-ping`).
- The inventory + vault + SSH-key wiring works end-to-end.
- `apt install` tasks in the `common` and `docker` roles run to completion.
- File/template rendering in the `vpn` role (compose.yml, nginx conf, v2ray config) produces valid files.

## What this does NOT validate

- **systemd services** — no systemd in the container. Tasks like `systemctl enable docker` will fail. Run these with `--skip-tags systemd` or skip the common/docker roles entirely when testing locally.
- **Docker-in-Docker** — the `docker` role installs docker-ce but `dockerd` won't run without dind. The vpn/letsencrypt roles depending on `docker compose up` will therefore fail.
- **Let's Encrypt bootstrap** — requires real DNS + port 80 reachable from LE's servers. Always skip in test mode.
- **Actual network traffic through V2Ray** — no VMess client connected; no WebSocket negotiation.

For full end-to-end validation, stand up a throwaway Ubuntu 22.04 VM on Azure/GCP/Hetzner ($5/mo, destroy after testing) and run `just deploy` against it.

## Usage

```bash
# First-time setup — generates a local test SSH keypair (gitignored):
just test-setup

# Bring up the container and test ansible reachability:
just test-up
just test-ping                    # expects "pong"

# Tear down:
just test-down
```

## Files

- `Dockerfile` — Ubuntu 22.04 + openssh-server + sudo + ansible user with passwordless sudo + `ssh/id_ed25519.pub` injected as `authorized_keys`.
- `docker-compose.test.yml` — builds the image and binds SSH to `127.0.0.1:2222`.
- `ssh/id_ed25519{,.pub}` — test-only SSH keypair, generated by `just test-setup`, private key gitignored.
- `../ansible/inventory/test.ini` — points Ansible at `127.0.0.1:2222` with the test key.

## Running parts of the playbook

After `just test-up`:

```bash
cd ansible
# Dry run the common role (expect systemd tasks to fail):
ansible-playbook -i inventory/test.ini playbooks/site.yml --tags common --check

# Render templates without executing handlers (works):
ansible-playbook -i inventory/test.ini playbooks/deploy.yml --check
```

To run the full playbook against the container, you'd need to skip systemd / docker-compose tasks — that's more involved than this MVP supports. For that level of fidelity, use a real VM.
