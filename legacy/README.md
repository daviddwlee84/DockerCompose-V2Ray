# Legacy — pre-IaC deploy flow

This folder preserves the files that comprised the original shell-script + Docker Compose deploy flow, before the Ansible-based IaC refactor.

Nothing here is used by the current deploy path. It exists as an **archival reference**:

- `scripts/` — original bootstrap scripts
  - `install_docker.sh` — tried to install both rootful *and* rootless Docker (conflicting; the rootless step would break port 80/443 binding). Replaced by `scripts/install_docker.sh` at the repo root, which is rootful-only.
  - `initial_https.sh` — shell wrapper that `sed`-substituted domain + email into `init-letsencrypt.sh` and the nginx config. Replaced by Ansible roles.
  - `init-letsencrypt.sh` — certbot bootstrap: created a dummy cert, started nginx, ran Let's Encrypt HTTP-01 challenge, reloaded nginx. Replaced by the `letsencrypt` Ansible role.
- `compose/` — original Docker Compose files
  - `docker-compose.yml` — Nginx + Certbot + V2Ray, with TLS via Let's Encrypt. Replaced by the templated compose file generated by Ansible.
  - `docker-compose-v2ray-without-tls.yml` — bare V2Ray on port 80 (no TLS). Retired: the refactor is TLS-always.
- `data/` — original config files that shipped with hardcoded values
  - `nginx/conf.d/v2ray.conf` — had `your_domain` placeholders that were `sed`-substituted at deploy time.
  - `v2ray/config.json` — had the hardcoded default UUID `bae399d4-13a4-46a3-b144-4af2c0004c2e`.
  - Both replaced by Jinja2 templates under the `vpn` Ansible role.
- `clients/`
  - `setup_linux_clash_client_old.sh` — superseded by `client/setup_linux_clash_client.sh`.
  - `ui_pages_old/` — older YACD build output; the current build is at `client_docker/ui_pages/`.
- `example/`
  - `clash_for_windows.yml` — original 40KB client config template with the hardcoded default UUID baked in.
